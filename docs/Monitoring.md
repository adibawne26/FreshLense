# Monitoring Guide

## Overview

FreshLense 2.0 includes a complete observability stack for monitoring application health, infrastructure performance, centralized logging, and alerting.

The monitoring solution is built around Prometheus, Grafana, Loki, Promtail, Alertmanager, and Node Exporter, providing real-time visibility into both the application and the underlying infrastructure.

---

# Monitoring Architecture

![Monitoring Stack](images/system-architecture.png)

The monitoring architecture consists of five major layers:

1. Metrics Collection
2. Visualization
3. Log Aggregation
4. Alerting
5. Infrastructure Monitoring

---

# Monitoring Stack

| Component | Purpose |
|------------|----------|
| Prometheus | Metrics Collection |
| Grafana | Dashboard Visualization |
| Loki | Centralized Log Storage |
| Promtail | Log Collection |
| Alertmanager | Alert Routing |
| Node Exporter | System Metrics |

---

# Prometheus

Prometheus continuously scrapes metrics exposed by the FreshLense backend.

Metrics endpoint:

```
http://localhost:8000/metrics
```

Scrape interval is configured inside:

```
monitoring/prometheus.yml
```

---

# Custom Application Metrics

FreshLense exposes custom Prometheus metrics for crawler performance.

## Crawl Requests

```
freshlense_crawl_requests_total
```

Tracks the total number of crawl requests executed.

---

## Successful Crawls

```
freshlense_crawl_success_total
```

Tracks successful crawl executions.

---

## Failed Crawls

```
freshlense_crawl_failure_total
```

Tracks failed crawl attempts.

---

## Crawl Duration

```
freshlense_crawl_duration_seconds
```

Histogram representing crawler execution time.

---

# HTTP Metrics

The backend also exposes automatic FastAPI metrics including:

- Request Count
- Response Status Codes
- Request Duration
- Response Size
- Request Size

These metrics are collected using Prometheus FastAPI Instrumentator.

---

# Node Exporter

Node Exporter provides host-level metrics including:

- CPU Usage
- Memory Usage
- Disk Utilization
- Filesystem Statistics
- Network Statistics
- Load Average

These metrics help monitor infrastructure health.

---

# Grafana

Grafana visualizes metrics collected by Prometheus.

Typical dashboards include:

- Crawl Requests
- Crawl Success Rate
- Crawl Failures
- Crawl Duration
- HTTP Requests
- System Resources

Grafana connects directly to Prometheus as its primary datasource.

---

# Loki

Loki provides centralized log aggregation.

Application logs from Docker containers are collected and indexed without requiring full-text indexing.

Log sources include:

- Backend
- Frontend
- Prometheus
- Grafana
- Alertmanager
- Node Exporter

---

# Promtail

Promtail continuously collects Docker container logs and forwards them to Loki.

Configuration:

```
monitoring/promtail/promtail-config.yml
```

---

# Alertmanager

Alertmanager receives alerts generated by Prometheus.

Responsibilities include:

- Alert grouping
- Alert deduplication
- Notification routing
- Alert silencing

Alert rules are configured in:

```
monitoring/rules/
```

---

# Health Monitoring

FreshLense exposes a dedicated health endpoint.

```
GET /health
```

Returns:

- Backend Status
- Database Status
- Service Availability

Container health checks are also configured for:

- Frontend
- Backend

Docker automatically monitors service availability.

# Monitoring Summary

FreshLense implements a production-ready observability solution by combining metrics, centralized logging, dashboards, infrastructure monitoring, and automated alerting.

This monitoring stack provides complete visibility into application behaviour, crawler performance, container health, and infrastructure utilization, enabling proactive system monitoring and faster issue diagnosis.
